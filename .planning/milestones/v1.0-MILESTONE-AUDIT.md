## Tech Debt

### Accepted Deviations

#### 1. PER-01: Zero-Allocation → Minimized Allocation
**Phase:** 01-foundation  
**Impact:** MEDIUM - Performance not guaranteed for embedded/high-frequency scenarios

**Decision:** Buffer and Message use `std::string` internally (STATE.md line 84: "可用性优先").

**Rationale:**
- SSO handles most cases without allocation
- Pre-allocated 4096 bytes minimizes allocations
- Better usability and safety trade-off accepted

**Mitigation:** Documented in REQUIREMENTS.md as requirement relaxation.

#### 2. PER-02: Inline Optimizations (Partial)
**Phase:** 01-foundation  
**Impact:** LOW

**Current State:**
- Key functions marked `inline` ✅
- `error_message()`, Buffer methods inline ✅
- No `__attribute__((always_inline))` hints

**Mitigation:** Sufficient for current performance targets (8-11 MB/s debug, >100 MB/s expected release).
